# Howrse Manager

**Howrse Manager** — Tampermonkey-приложение для Ловади / Howrse.

Проект создаётся как будущий умный менеджер табуна: с красивым интерфейсом, модульной архитектурой, настройками, логом, анализом лошади и постепенным добавлением автоматизации.

## Текущая версия

`v0.2.0` — анализ страницы и текущей лошади.

## Что есть сейчас

- плавающее окно приложения на `www.lowadi.com`;
- современный интерфейс со страницами;
- светлая, тёмная и автоматическая тема;
- сохранение настроек в браузере;
- кнопки `Старт`, `Пауза`, `Стоп`;
- базовый лог событий;
- раздел разработчика;
- определение типа страницы:
  - список лошадей;
  - страница лошади;
  - КСК;
  - соревнования;
  - неизвестная страница;
- первичный анализ текущей лошади:
  - имя;
  - энергия;
  - здоровье;
  - мораль;
  - возраст;
  - пол;
  - наличие кнопки следующей лошади;
- показ найденных селекторов и сырого текста в разделе `Разработчик`;
- заготовка архитектуры:
  - `Application`;
  - `EventBus`;
  - `Storage`;
  - `Logger`;
  - `SettingsManager`;
  - `StateManager`;
  - `DelayManager`;
  - `RouteManager`;
  - `SelectorManager`;
  - `HorseParser`;
  - `LowadiAdapter`;
  - `UIManager`.

## Чего пока нет

Скрипт пока **не выполняет игровые действия**:

- не кликает по кнопкам;
- не прогоняет табун;
- не тренирует лошадей;
- не записывает в КСК.

`v0.2` нужна, чтобы безопасно проверить, насколько хорошо скрипт видит страницу и данные лошади.

## Установка для теста

1. Установить расширение Tampermonkey.
2. Открыть файл [`howrse-manager.user.js`](./howrse-manager.user.js).
3. Скопировать содержимое файла в новый userscript Tampermonkey.
4. Сохранить скрипт.
5. Открыть `https://www.lowadi.com/`.

## Как тестировать v0.2

1. Открыть страницу списка лошадей.
2. Проверить, что в разделе `Разработчик` тип страницы определяется как `Список лошадей`.
3. Открыть страницу конкретной лошади.
4. Нажать `Анализ`.
5. Проверить, какие данные найдены:
   - имя;
   - энергия;
   - здоровье;
   - мораль;
   - возраст;
   - пол;
   - кнопка следующей лошади.
6. Если что-то не найдено, прислать скрин раздела `Разработчик`.

## Планы следующих версий

- `v0.4` — базовый уход.
