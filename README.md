# Howrse Manager

**Howrse Manager** — Tampermonkey-приложение для Ловади / Howrse.

Проект создаётся как будущий умный менеджер табуна: с красивым интерфейсом, модульной архитектурой, настройками, логом, анализом лошади и постепенным добавлением автоматизации.

## Текущая версия

`v0.3.0` — гибридный табунный режим.

## Что есть сейчас

- плавающее окно приложения на `www.lowadi.com`;
- современный интерфейс со страницами;
- светлая, тёмная и автоматическая тема;
- сохранение настроек в браузере;
- кнопки `Старт`, `Пауза`, `Продолжить`, `Стоп`;
- мягкая остановка после текущей лошади;
- базовый лог событий;
- раздел разработчика;
- определение типа страницы:
  - список лошадей;
  - страница лошади;
  - КСК;
  - соревнования;
  - неизвестная страница;
- первичный анализ текущей лошади:
  - ID;
  - имя;
  - энергия;
  - здоровье;
  - настроение;
  - возраст;
  - пол;
  - наличие кнопки следующей лошади;
- гибридный табунный режим:
  - анализирует текущую лошадь;
  - отмечает её обработанной;
  - ждёт случайную задержку;
  - нажимает кнопку следующей лошади;
  - после загрузки новой страницы продолжает работу;
  - останавливается при лимите, паузе, стопе, мягкой остановке или повторе лошади;
- заготовка архитектуры:
  - `Application`;
  - `EventBus`;
  - `Storage`;
  - `Logger`;
  - `SettingsManager`;
  - `StateManager`;
  - `DelayManager`;
  - `RouteManager`;
  - `SelectorManager`;
  - `HorseParser`;
  - `HerdRunner`;
  - `LowadiAdapter`;
  - `UIManager`.

## Важно

`v0.3` уже может нажимать кнопку следующей лошади.

Но скрипт пока **не выполняет уход и игровые действия**:

- не чистит;
- не кормит;
- не поит;
- не отправляет спать;
- не тренирует;
- не записывает в КСК.

Эта версия нужна, чтобы проверить маршрут табуна: текущая лошадь → следующая лошадь.

## Установка для теста

1. Установить расширение Tampermonkey.
2. Открыть файл [`howrse-manager.user.js`](./howrse-manager.user.js).
3. Скопировать содержимое файла в новый userscript Tampermonkey.
4. Сохранить скрипт.
5. Открыть `https://www.lowadi.com/`.

## Как тестировать v0.3

1. Открыть страницу конкретной лошади.
2. В настройках прогона поставить небольшой лимит, например `3`.
3. Нажать `Старт`.
4. Проверить, что скрипт:
   - отметил текущую лошадь;
   - перешёл к следующей;
   - продолжил после загрузки страницы;
   - остановился по лимиту.
5. Если переход не сработал — прислать скрин раздела `Разработчик`.
